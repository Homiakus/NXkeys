# Диагностика NXKeys

## 1. Базовый порядок проверки

1. Закройте NX при обновлении DLL и MenuScript.
2. Проверьте JSON-профиль.
3. Выполните план без применения.
4. Проверьте установленный пакет.
5. Запустите NX через управляемую обёртку.
6. Проверьте состояние Bridge и его очереди.
7. Повторите конкретную команду в правильном модуле NX.

```powershell
$studio = "$env:LOCALAPPDATA\NXKeys\managed\NX2512.6000\NX2512_HotkeyStudio.exe"

& $studio validate --config .\config\nx2512-pro-hybrid.json
& $studio plan --config .\config\nx2512-pro-hybrid.json
& $studio health --config .\config\nx2512-pro-hybrid.json
& $studio bridge-status
```

## 2. NX не видит NXKeys

Проверьте:

- NX запущен через `launch-nx2512-with-nxkeys.cmd`;
- существует управляемый `custom_dirs.dat`;
- в `custom\startup` и `custom\application` находятся ожидаемые файлы;
- путь к NX в профиле корректен;
- установленный пакет не отличается по хешу от собранного;
- MenuScript содержит правильную `VERSION`.

## 3. Ошибка VERSION в `.men`

Для `.men` требуется:

```text
VERSION 139
```

После исправления закройте NX, пересоберите/переустановите пакет и выполните `health`.

## 4. Ошибка VERSION в `.tbr` или `.rtb`

Для toolbar/ribbon требуется:

```text
VERSION 170
```

Файлы `.tbr/.rtb` должны оставаться тонким слоем размещения. Определения кнопок и действий размещаются в `.men`.

## 5. Установщик не может скопировать Bridge DLL

Причина обычно в том, что NX продолжает удерживать DLL.

1. Закройте все окна NX.
2. Проверьте процессы `ugraf.exe` и связанные процессы NX.
3. Завершите оставшиеся процессы только после сохранения данных.
4. Повторите сборку и установку.

## 6. Bridge показывает OFFLINE

Когда NX закрыт, это нормально.

Когда NX открыт:

- убедитесь, что NX запущен через NXKeys wrapper;
- проверьте `%LOCALAPPDATA%\NXKeys\bridge\status.json`;
- проверьте `context.json`;
- откройте журналы в `%LOCALAPPDATA%\NXKeys\logs`;
- проверьте наличие `NX2512_CommandBridge.dll` в managed custom directory.

## 7. Bridge показывает STALE

Control Center считает контекст свежим около 10 секунд после `updated_utc`.

Возможные причины:

- таймер Bridge не работает;
- библиотека не загрузилась;
- `context.json` заблокирован или повреждён;
- NX завис;
- установлена старая DLL Bridge.

Закройте NX, переустановите Bridge и повторите запуск через wrapper.

## 8. Команда попала в `failed`

Проверьте результат в:

```text
%LOCALAPPDATA%\NXKeys\bridge\failed
```

Типовые причины:

- `BUTTON ID` отсутствует в этой сборке NX;
- команда недоступна в текущем приложении;
- кнопка нечувствительна без выбранного объекта;
- активен несовместимый диалог;
- отсутствует лицензия;
- команда была переименована или удалена;
- запрос устарел или повреждён.

Сначала попробуйте вручную найти и выполнить ту же команду в NX. Затем сверяйте точный ID с каталогом целевой установки.

## 9. Control Center показывает неизвестный выбор

`Selection: unknown` или значение `-1` означает, что установленная версия Bridge не передала число выбранных объектов. Это не равнозначно нулевому выбору.

В таком состоянии Control Center не должен использовать отсутствие данных как строгую причину блокировки. Проверяйте команду непосредственно в NX.

## 10. Control Center не находит HotkeyStudio

Разместите `NX2512_ControlCenter.exe` и его зависимости рядом с `NX2512_HotkeyStudio.exe` либо в ожидаемой соседней папке.

Control Center запускает Studio по относительному пути и не выполняет глобальный поиск по системе.

## 11. API-каталог не найден

Укажите каталог:

```powershell
.\dist\control-center\NX2512_ControlCenter.exe `
  --config .\config\nx2512-pro-hybrid.json `
  --catalog "D:\NX2512_Full_Function_API_Catalog_YYYYMMDD_HHMMSS"
```

Либо задайте:

```powershell
$env:NXKEYS_CATALOG_DIR = "D:\NX2512_Full_Function_API_Catalog_YYYYMMDD_HHMMSS"
```

В каталоге должен присутствовать хотя бы один файл:

```text
04_nxopen_members.csv
06_ui_commands_buttons.csv
```

## 12. Поиск API даёт слабые результаты

Текущий поиск основан на токенах и небольшом словаре. Используйте:

- английское имя команды;
- имя Builder/Manager;
- точный `UG_...`;
- название класса NXOpen;
- фрагмент UFUN-функции.

Пример:

```text
extrude feature builder
NXOpen.Features.ExtrudeBuilder
UF_MODL
UG_MODELING_EXTRUDE
```

Crosswalk является кандидатом и требует ручной проверки сигнатуры.

## 13. Профиль не сохраняется

Проверьте:

- существование каталога;
- права записи;
- отсутствие read-only атрибута;
- корректность JSON;
- не открыт ли файл другой программой;
- не используется ли путь из защищённого каталога установки.

Для рабочего профиля рекомендуется каталог пользователя, а не `Program Files`.

## 14. Неоднозначная команда

Если в `resolution-report.md` указано несколько кандидатов, заполните точный ID:

```json
{
  "command": {
    "id": "UG_EXACT_BUTTON_ID",
    "name": "Читаемое имя"
  }
}
```

Не снижайте пороги нечёткого поиска только ради прохождения валидации.

## 15. Конфликт горячих клавиш

Проверьте:

- назначения в `Ctrl+1 → Customize → Keyboard`;
- корпоративную роль;
- другие custom directories;
- сгенерированный `.men`;
- значение `clear_detected_conflicts`.

NXKeys не может полностью проверить ускорители внутри непрозрачного `.mtx`.

## 16. Восстановление

Список резервных копий:

```powershell
& $studio backups --config .\config\nx2512-pro-hybrid.json
```

Последняя резервная копия:

```powershell
& $studio restore --config .\config\nx2512-pro-hybrid.json
```

Конкретный манифест:

```powershell
& $studio restore `
  --config .\config\nx2512-pro-hybrid.json `
  --manifest "$env:LOCALAPPDATA\NXKeys\backups\YYYYMMDD_HHMMSS.mmm\manifest.json"
```

Если хеш целевого файла изменился после применения, сначала сохраните текущую версию и выясните источник изменений.

## 17. Что приложить к отчёту об ошибке

- версия Windows;
- точная версия NX;
- используемая роль и приложение NX;
- команда/последовательность;
- результат `validate`, `plan`, `health`;
- содержимое соответствующего файла из `bridge/failed`;
- последние строки журнала Bridge;
- обезличенный фрагмент профиля;
- сведения о лицензии для проблемного приложения.
