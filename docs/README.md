## Интерактивная карта

Файл `command-tree.html` является автономной HTML-диаграммой без внешних библиотек.

Рекомендуемый запуск из корня репозитория:

```powershell
py -m http.server 8080
```

Затем откройте:

```text
http://localhost:8080/docs/command-tree.html
```

В HTTP-режиме карта автоматически загружает:

```text
config/nx2512-pro-hybrid.json
config/nx2512-state-machines.json
```

При прямом открытии через `file://` загрузите оба JSON кнопками на странице или перетащите их в окно браузера.

## Источник истины

Настраиваемыми источниками истины являются:

```text
config/nx2512-pro-hybrid.json
config/nx2512-state-machines.json
```

Сгенерированные `.men`, `.tbr`, `.rtb`, отчёты разрешения, планы радиальных меню и HTML-представления являются производными артефактами и не должны расходиться с каноническими JSON.

CI выполняет `scripts/validate-command-tree.mjs`, проверяет полноту канонической карты и обязательные механизмы динамической загрузки/визуализации HTML.

## Обозначение статуса функций

- **реализовано** — функция присутствует в исходном коде и проходит предусмотренные проверки;
- **зависит от NX** — результат определяется лицензией, ролью, локализацией или активным приложением;
- **кандидатное сопоставление** — найден вероятный UI/API-аналог, требующий проверки;
- **целевая функция** — архитектурное направление, которое ещё нельзя считать полностью готовым.

## Версии

Документация относится к ветке `main` и Siemens NX 2512. Для другой версии NX необходимо повторно проверить `BUTTON ID`, MenuScript, NXOpen-сборки, роли, доступность приложений и контекстные guards.
