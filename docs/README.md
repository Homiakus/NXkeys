# Документация NXKeys

Документация относится к адаптивной архитектуре NXKeys для Siemens NX 2512:

- 12 фиксированных базовых сочетаний;
- 14 контекстных модулей;
- восемь команд активного модуля на сетке `QWE / A·D / ZXC`;
- автоматический выбор набора по контексту Command Bridge;
- DFA/HFSM, guards, подтверждение и транзакционная очередь.

## Интерактивная карта

Из корня репозитория выполните:

```powershell
py -m http.server 8080
```

Затем откройте:

```text
http://localhost:8080/docs/command-tree.html
```

Страница автоматически читает:

```text
config/nx2512-pro-hybrid.json
config/nx2512-state-machines.json
```

При открытии через `file://` оба JSON можно загрузить кнопками или перетащить в браузер.

## Источник истины

Редактируются только два JSON-файла:

```text
config/nx2512-pro-hybrid.json
config/nx2512-state-machines.json
```

Производные последовательности Leader строятся из `modules[].command_sets[].commands` при загрузке профиля и не сериализуются. Сгенерированные `.men`, `.tbr`, `.rtb`, CMD-файлы, отчёты и package manifest вручную не редактируются.

## Проверка документации

```powershell
node .\scripts\validate-command-tree.mjs
```

Валидатор сверяет базовые сочетания, 14 модулей, 112 команд, сетку ввода, HTML-представления и отсутствие удалённой подсистемы.

## Статусы

- **реализовано** — функция присутствует в коде и проверяется CI;
- **зависит от NX** — результат зависит от роли, лицензии, локализации или контекста;
- **кандидатное сопоставление** — UI/API-аналог требует проверки;
- **интеграционно проверено** — команда подтверждена внутри целевой сборки NX.
