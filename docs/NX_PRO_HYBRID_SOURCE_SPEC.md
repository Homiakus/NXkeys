# Спецификация NX Adaptive Modules 2512

## Цель

Покрыть высокочастотный рабочий процесс Siemens NX минимальным моторным алфавитом:

- 12 базовых прямых сочетаний;
- один Leader;
- восемь физических клавиш;
- контекстно выбранный набор текущего приложения.

## Канонические модули

| ID | Приложение | Внутренний префикс |
|---|---|---|
| `modeling` | Modeling | `M` |
| `sketch` | Sketch | `S` |
| `assembly` | Assembly | `A` |
| `drafting` | Drafting | `D` |
| `pmi` | PMI | `P` |
| `surface` | Surface | `U` |
| `sheet_metal` | Sheet Metal | `H` |
| `manufacturing` | CAM / Manufacturing | `C` |
| `simulation` | CAE / Simulation | `X` |
| `routing` | Routing | `G` |
| `mold` | Mold / Tooling | `O` |
| `reuse` | Reuse / Templates | `R` |
| `inspect_view` | Inspect / View | `V` |
| `selection_object` | Selection / Object | `F` |

Внутренний префикс нужен DFA и не вводится пользователем.

## Смысловая геометрия клавиатуры

| Физическая клавиша | Слот | Назначение |
|---|---|---|
| `W` | `N` | начать, создать, открыть основной объект |
| `E` | `NE` | следующий главный шаг |
| `D` | `E` | добавить объект или зависимость |
| `C` | `SE` | преобразовать или заменить |
| `X` | `S` | завершить, обработать или удалить |
| `Z` | `SW` | удалить, уменьшить или ослабить |
| `A` | `W` | структура, связь, паттерн |
| `Q` | `NW` | проверить, измерить, открыть сервис |

Эта геометрия одинакова во всех модулях, поэтому навык переносится между Modeling, Sketch, Sheet Metal, Drafting и остальными приложениями.

## Правила наполнения модуля

1. Ровно восемь команд.
2. Команда должна иметь точный `BUTTON ID`.
3. Наиболее частые операции занимают `W/E/D/C`.
4. Операции завершения и удаления занимают `X/Z`.
5. Структурные и диагностические действия занимают `A/Q`.
6. Контекстная операция помечается `requires_selection`.
7. Разрушительная операция помечается `destructive` и `confirm_before_execute`.
8. Команда не дублируется прямым сочетанием, кроме базового системного минимума.

## Определение покрытия

Целевое покрытие относится не к количеству всех команд NX, а к типовым сценариям:

- создание геометрии;
- редактирование и преобразование;
- сборки;
- чертежи и PMI;
- поверхности и листовой металл;
- CAM и CAE;
- routing и tooling;
- повторное использование;
- просмотр и выбор.

Команда считается готовой к эксплуатации, когда:

1. присутствует в каноническом модуле;
2. имеет точный `BUTTON ID`;
3. проходит проверку профиля;
4. корректно разрешается в каталоге целевой версии;
5. guards отражают требования контекста;
6. Bridge выполняет её внутри нужного приложения;
7. опасная операция требует подтверждение;
8. результат проверен на целевой рабочей станции.

## Поведение при смене модуля

- контекст Bridge обновляется;
- `AdaptiveModuleResolver` выбирает новый `ModuleConfig`;
- открытый HUD перестраивается;
- search очищается и ограничивается новым набором;
- прежние команды перестают быть видимыми;
- незавершённый ввод отменяется;
- запрос на смену приложения считается завершённым только после новой ревизии контекста.

## Расширение профиля

Добавление нового модуля требует:

- уникального `id`;
- точных `nx_application_ids`;
- команды переключения;
- уникального внутреннего префикса;
- восьми команд;
- policy guards;
- теста разрешения контекста;
- проверки HTML-карты и CI.
