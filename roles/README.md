# Экспортированные роли Siemens NX 2512

В этот каталог помещаются только проверенные роли `.mtx`, экспортированные из целевой версии Siemens NX 2512.

## Назначение

NXKeys не декодирует и не изменяет внутреннюю структуру `.mtx`. Роль рассматривается как непрозрачный артефакт, который можно:

- сохранить в репозитории;
- проверить на тестовой рабочей станции;
- скопировать в целевой каталог;
- восстановить из резервной копии.

## Подготовка роли

1. Откройте NX 2512.
2. Настройте горячие клавиши и радиальные меню штатными средствами NX.
3. Проверьте команды в нужных приложениях и с нужными лицензиями.
4. Экспортируйте роль `.mtx`.
5. Закройте NX.
6. Поместите проверенный файл в `roles/`.
7. Укажите его в JSON-профиле.

Пример:

```json
{
  "role_deployment": {
    "enabled": true,
    "source_mtx": "roles\\nx2512-tested-role.mtx",
    "target_directory": "",
    "target_name": "",
    "set_as_default": false,
    "default_role_env": "UGII_DEFAULT_ROLE"
  }
}
```

## Правила безопасности

- не используйте роль из другой основной версии NX без проверки;
- не заменяйте корпоративную роль без резервной копии;
- не включайте `set_as_default`, пока тестовый запуск не подтверждён;
- не храните в репозитории роли с конфиденциальными корпоративными путями или настройками;
- не пытайтесь редактировать `.mtx` текстовым или бинарным редактором;
- после обновления NX повторно проверьте Application Radials и Object Radials.

## Что следует хранить в Git

Допускается хранить только намеренно распространяемые и обезличенные роли, для которых зафиксированы:

- версия NX;
- дата экспорта;
- назначение;
- поддерживаемые модули;
- результат ручной проверки;
- связанные JSON-профили.

Рекомендуемое имя:

```text
nx2512-<назначение>-<версия_роли>.mtx
```

Пример:

```text
nx2512-pro-hybrid-v3.mtx
```

Подробная модель безопасности: [../docs/SAFETY_MODEL.md](../docs/SAFETY_MODEL.md).
