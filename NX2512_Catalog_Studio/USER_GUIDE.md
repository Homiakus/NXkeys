# Руководство пользователя

## Рекомендуемый первый запуск

1. Откройте страницу **Состав** и оставьте включёнными все четыре источника.
2. На странице **Пути** проверьте папку результата.
3. Если используются корпоративные каталоги, добавьте их в список дополнительных путей.
4. Откройте **Профили**, задайте название и сохраните JSON.
5. Перейдите в **Запуск** и нажмите **Запустить**.
6. После завершения откроется страница **Результаты**.

## Быстрые профили

### Только UI-команды

Оставить:

- UI-команды;
- пути;
- итоговый отчёт.

Отключить NX Open, UFUN и crosswalk.

### Только программный NX Open API

Оставить:

- managed API;
- assemblies;
- namespace;
- types;
- members;
- entry points.

### Полный корпоративный аудит

Включить всё и добавить:

- `UGII_SITE_DIR`;
- `UGII_GROUP_DIR`;
- пользовательские custom directories;
- каталоги собственных приложений NX.

## Остановка

Кнопка **Остановить** выставляет CancellationToken. Уже записанные таблицы остаются в каталоге результата; незавершённый этап прекращается при ближайшей безопасной точке.

## Порог сопоставления

- `MinimumCandidateScore` — какие кандидаты вообще попадут в таблицу;
- `StrongCandidateScore` — с какого значения команда считается имеющей сильного кандидата;
- `CandidateLimit` — максимальное число кандидатов на один BUTTON ID.

Даже высокий результат следует проверять записанным NX Journal.
